About

To add a check, edit /etc/gogios/checks.json and add a new block (or replace on of the example ones). The format is: 

{
    "title": "The title of the check",
    "command": "The command to run. For example 'echo 0'. Note that these commands will not be forced to exit, they'll run until they end.",
    "expected": "The expected output of that command.",
    "good": false,
    "asof": ""
}

To easily convert an nmap scan to a framework for a Gogios check file, you can use the gogios-parse-nmap script. Run an nmap scan with -oX scan.xml, and then: 

      gogios-parse-nmap -f scan.xml -o scan.json

And then take the checks that you want from that output and add it to your checks.json.

Plugins that I have built are stored in /usr/lib/gogios/plugins, and examples of the usage is available by running them with the --help flag, or by looking in /etc/gogios/example.json.

Commands will be run as the gogios user, who by default has no root privileges. If possible, I suggest keeping it that way.
